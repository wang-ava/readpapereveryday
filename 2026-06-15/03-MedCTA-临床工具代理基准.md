# MedCTA: A Benchmark for Clinical Tool Agents

MedCTA 指向医疗场景中真正的瓶颈：模型不是只要识别影像或问答准确，而是必须能在多模态临床上下文里规划工具链、稳定调用并交付可审计的诊疗路径。它把“会说”升级为“会执行”。

- **论文标题**：MedCTA: A Benchmark for Clinical Tool Agents
- **作者/机构**：Tajamul Ashraf, Hyewon Jeong, Fida Mohammad Thoker, Bernard Ghanem（KAUST；MIT）
- **发布日期/版本日期**：2026-06-10（`v1`）
- **主题标签**：#AI4S #HealthcareAI #Multimodal #ToolUse #Benchmark #Agent
- **论文链接**：<https://arxiv.org/abs/2606.11702>
- **PDF 链接**：<https://arxiv.org/pdf/2606.11702v1>
- **项目/代码/数据链接（如可得）**：<https://ivul-kaust.github.io/MedCTA/>
- **核心问题**：现有医疗评测大多停留在感知或单轮问答，缺少真实临床工作流所需的工具路由、参数构造、证据串联和执行稳定性评估。
- **方法概要**：
  1. 构建 MedCTA：107 条真实临床任务，基于医学影像、病理切片、报告文本构建多模态输入。
  2. 将工具使用嵌入任务定义：任务不显式给出工具顺序，要求模型自主规划。
  3. 部署 5 类可执行工具，覆盖报告读取、区域属性提取、OCR、网络检索、计算等。
  4. 设计过程级评价：包括工具选择正确性、参数合法性、执行稳定性、轨迹完整度与结论质量。
- **主要贡献**：
  1. 提供首个面向临床工具代理的隐含路由 benchmark，强调可执行与可追踪。
  2. 通过“step-implicit query + hidden sufficient tool subset”将评估重心从感知准确度转为行动链路正确性。
  3. 给出现成评测脚本与分析口径，便于复现和扩展。
- **关键实验或结果**：
  - 评估 18 个开源/闭源多模态模型后发现：主流模型在多步临床工具使用上仍脆弱。
  - 主要失败模式集中在协议选择错误、过早结束和工具调用错误。
  - 即便给定金标准工具路由，性能仍非完美，说明动作规划仍是瓶颈。
- **适合关注的原因**：
  1. AI4S 场景中的医学任务需要“可审计行为链”，这类基准比传统 VQA 更贴近落地。
  2. 标注过程兼顾放射/病理与流程真实性，能暴露真实部署中的风险点。
  3. 数据结构化程度高，适合做持续迭代评估。
- **局限性或待验证点**：
  1. 基准规模虽较大但仍集中于特定医疗机构与任务模板，地域外推需验证。
  2. 工具生态仍局限于少量可执行接口，真实医院系统接入复杂。
  3. 对模型幻觉与人机协同安全边界的长期追踪仍不足。
- **对后续研究/应用的启发**：可作为 AI4S 临床 agent 的“安全回归基准”，先用 MedCTA 约束工具行为，再逐步接入 EHR、RIS/PACS 与知识图谱。
- **Obsidian 快速浏览的一句总结**：MedCTA 让临床 AI 评测从“会不会看”升级为“会不会做、做得对不对、执行稳定不稳定”。

## 标准化研究框架
- **Research question：** 面向临床任务，如何有效评估多模态模型在工具规划与执行过程中的可靠性？
- **Literature：** 对接医疗 LLM、tool-use agent、clinical QA 基准三类文献，核心区别在于把可执行轨迹放到主评估对象。
- **Theory：** 等价于过程控制框架：诊断决策不是单次输出，而是带状态转移和动作执行的序列决策问题。
- **Hypotheses：** 在固定工具集合下，模型的临床性能主要由工具路由与参数有效性决定，而非纯视觉识别或语言表述能力。
- **Method：** 通过 107 条 step-implicit 临床任务建立任务库；评估多模型执行轨迹并拆分工具级指标与最终结果级指标。
- **Data and Analysis：** 基于公开的 MedCTA 任务与五种工具，统计工具选择准确率、参数有效率、执行稳定性、轨迹保真度和结果质量。
- **Findings：** 在现有前沿模型中，多步工具代理能力远未达到稳健标准；即使有黄金工具序列，仍存在显著剩余误差。
- **Conclusion：** MedCTA 对医疗 AI 4S 的贡献在于把“推理能力”与“执行能力”分离评估，后续系统建设应把安全、稳定与可审计性置于同等优先级。
