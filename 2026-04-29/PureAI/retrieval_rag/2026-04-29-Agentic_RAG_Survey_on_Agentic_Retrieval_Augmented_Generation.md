# Agentic Retrieval-Augmented Generation: A Survey on Agentic RAG

## 背景知识：从 RAG 到 Agentic RAG，发生了什么？

### 先说 RAG（Retrieval-Augmented Generation，检索增强生成）

LLM 有一个根本问题：**知识截止日期**。GPT-4 的训练数据截止到某个时间，之后的事它不知道。另外，LLM 会"幻觉"——自信地说出错误内容。

RAG 的解决方案：
1. 给 LLM 配备一个**外部知识库**（文档、数据库、网页等）
2. 用户提问时，先从知识库里**检索**相关内容
3. 把检索到的内容和问题一起输入 LLM，让它**基于事实**回答

简单类比：RAG 让 LLM 从"默写"变成了"开卷考试"，能查资料了。

### 什么是 Agentic RAG？

传统 RAG 是被动的、单步的：一问一答，检索一次，生成一次，结束。

但复杂问题需要**多步骤、自主决策**：
- "帮我分析一下 2024 年 A 公司的年报，对比 B 公司，判断哪家更值得投资"
- 这个问题需要：搜索 A 公司年报 → 读取关键数据 → 搜索 B 公司年报 → 对比分析 → 联系宏观经济背景 → 得出结论

**Agentic RAG** 让 RAG 系统具备"智能体（Agent）"能力：
- **自主规划**：决定需要检索什么、检索几次
- **工具调用**：不只是搜索文档，还能调用 API、执行代码、浏览网页
- **迭代推理**：根据每次检索结果决定下一步行动
- **自我反思**：判断已获得的信息是否足够，是否需要补充检索

## 怎么干的？

### Agentic RAG 的四种架构模式

**模式一：单智能体 RAG**
- 一个 LLM 同时承担规划、检索、推理的所有职责
- 简单但容易在复杂任务中迷失

**模式二：多智能体协作 RAG**
- 不同智能体负责不同分工：
  - 规划者智能体：分解任务
  - 检索者智能体：执行搜索
  - 验证者智能体：检查信息可靠性
  - 生成者智能体：整合输出
- 更强大，但协调成本更高

**模式三：层次化 RAG**
- 主智能体将任务分解给子智能体
- 子智能体并行工作，主智能体整合结果
- 适合大型复杂任务

**模式四：反思性 RAG（Self-RAG）**
- 模型在生成过程中自主判断是否需要检索
- 生成→自我评估→决定是否检索→继续生成
- 比固定触发检索更灵活

### 关键技术组件

**1. 查询改写（Query Rewriting）**
- 用户的原始问题可能模糊或不适合直接检索
- 智能体先把问题改写成更适合搜索的形式
- 例如："苹果的历史"→"苹果公司创始历史"+ "苹果水果起源"（消歧义）

**2. 自适应检索策略**
- 稀疏检索（BM25）：基于关键词匹配，速度快
- 密集检索（DPR）：基于语义相似度，准确性高
- 混合检索：结合两者优点

**3. 记忆机制**
- 短期记忆：当前对话的上下文
- 长期记忆：跨对话的用户偏好和历史

**4. 幻觉检测与自我纠正**
- 生成内容后，自动检测是否与检索到的文档矛盾
- 发现矛盾则触发重新检索和重新生成

## 核心结论

1. **Agentic RAG 显著扩展了传统 RAG 的能力边界**，使其能处理需要多步推理和多次检索的复杂任务。

2. **多智能体框架是处理复杂任务的有效方案**：分工合作优于单个全能智能体。

3. **自我反思机制对提升输出质量至关重要**：没有质量验证的 RAG 系统会"垃圾进垃圾出"。

4. **检索与推理的深度融合是未来方向**：不是"先检索后推理"，而是检索和推理互相驱动的循环。

---

## 局限性（Limitations）

1. **延迟增加**：多步检索和多智能体协调使得响应时间比单步 RAG 长很多，不适合实时场景。

2. **错误积累**：多步流程中任何一步的错误都可能被后续步骤放大，导致最终答案严重偏离。

3. **成本高**：每次 API 调用和每次检索都有成本，Agentic RAG 的运行成本远高于简单 RAG。

4. **可控性差**：智能体的自主决策有时难以预测，可能走向非预期的路径。

5. **评估困难**：如何衡量"多步检索+生成"的质量，没有统一的评估框架。
