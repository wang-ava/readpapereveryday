# Vision Mamba: Efficient Visual Representation Learning with Bidirectional State Space Model

**保存日期：** 2026-04-29
**论文链接：** https://arxiv.org/abs/2401.09417
**发表时间：** 2024年1月（arXiv）
**领域：** 纯 AI / 计算机视觉 / 高效架构 / 状态空间模型

---

## 背景知识：Transformer 在视觉领域有什么问题？

**Vision Transformer（ViT）** 是把 Transformer 用于图像识别的方法，把图像切成小块（patch），然后用 Transformer 处理这些块的序列。ViT 在精度上非常强，但有一个致命缺点：

**计算复杂度随序列长度二次方增长（O(n²)）**

什么意思？如果图像分辨率翻倍，patch 数量翻 4 倍，计算量翻 16 倍！这在处理高分辨率图像时代价极高。

而处理医学影像（整张CT切片）、卫星图像、高清视频等需要高分辨率的场景时，ViT 的计算成本让很多应用场景变得不可行。

**Mamba（2023年底）** 提出了解决方案：**状态空间模型（SSM，State Space Model）**，计算复杂度是线性的 O(n)。但 Mamba 最初是为文本序列设计的，图像有其特殊性——图像有二维空间结构，而文字是一维序列。

Vision Mamba 就是把 Mamba 成功带入图像领域的关键论文。

---

## 这篇论文干了什么？

Vision Mamba（简称 Vim）提出了一个**基于双向 Mamba 的视觉主干网络**，主要创新：

1. 让 Mamba 处理图像 patch 序列（解决一维→二维的问题）
2. 引入**双向（Bidirectional）**扫描，同时从左到右和从右到左扫描图像
3. 在速度和精度上都优于同规模的 ViT

---

## 怎么干的？

### 状态空间模型（SSM）原理

SSM 来自控制论，把序列处理建模为一个"状态机"：

```
给定输入序列 x(t)：
状态更新：h(t) = A * h(t-1) + B * x(t)
输出：y(t) = C * h(t)
```

其中 h 是隐藏状态（记忆），A、B、C 是可学习的参数矩阵。

**Mamba 的关键改进（Selective SSM）**：
- 让 A、B、C 矩阵根据**当前输入**动态变化（而非固定值）
- 这让模型能选择性地"记住"重要信息、"忘记"不重要信息
- 实现了线性复杂度但保持了强大的序列建模能力

### 图像处理的适配

**Patch 化**：
- 把 224×224 的图像切成 16×16 的 patch，得到 196 个 patch
- 每个 patch 变成一个向量（token）

**位置编码**：
- 加入 2D 位置编码，让模型知道每个 patch 在图像中的位置

**双向扫描**：
- 关键创新！Mamba 是单向的（像 RNN 一样），但图像需要全局感受野
- 解决方案：同时从左上→右下 和 右下→左上 两个方向扫描
- 两个方向的结果融合，模拟 Transformer 的双向注意力

### 完整架构

```
输入图像 → Patch Embedding → 
[Vim Block × N] → 分类头 → 输出
```

每个 Vim Block 包含：
- 双向 SSM（核心）
- 归一化层
- 残差连接

## 实验结果

| 模型 | ImageNet 精度 | 参数量 | 推理速度 |
|------|-------------|--------|---------|
| DeiT-S（Transformer）| 79.8% | 22M | 基准 |
| Vim-S（Vision Mamba）| 80.5% | 26M | **比 DeiT-S 快** |
| DeiT-B（Transformer）| 81.8% | 86M | - |
| Vim-B（Vision Mamba）| 81.9% | 98M | **比 DeiT-B 节省 67% 显存** |

- **更快**：在高分辨率（1248×1248）下，Vim 的速度和显存消耗远优于 ViT
- **更省内存**：处理高分辨率图像时，Vim 比同性能 ViT 节省约 2/3 的 GPU 内存

---

## 核心结论

1. **SSM 可以有效用于图像识别**，双向扫描策略成功弥补了 Mamba 单向性的缺陷。

2. **线性复杂度优势在高分辨率下最为显著**：图像越大，Vim 比 ViT 的优势越明显。

3. **参数效率高**：相同参数量下，Vim 的精度与 ViT 相当甚至略优。

4. **扩展性好**：Vim 的架构很适合处理医学影像、遥感图像等高分辨率任务。

---

## 局限性（Limitations）

1. **感受野不如 Attention 全局**：虽然双向扫描改善了全局感知，但理论上仍不如 Attention 机制的完全全局性。

2. **实现复杂度高**：Mamba 的高效实现依赖特定的 CUDA 内核，比 Transformer 更难部署和调试。

3. **在小图像上优势不明显**：对于标准 224×224 的 ImageNet 任务，Vim 的速度优势并不突出，只有高分辨率才能体现。

4. **尚未在大规模预训练中充分验证**：与 ViT 相比，Mamba 系架构的大规模预训练（类似 MAE、CLIP）研究还不够充分。
