# 论文标题：ReAct——让AI边想边做，推理与行动协同解决复杂问题

## 📄 论文基本信息
- **标题**：ReAct: Synergizing Reasoning and Acting in Language Models
- **作者**：Shunyu Yao 等（普林斯顿大学 / Google Brain）
- **发表**：ICLR 2023
- **论文链接**：https://arxiv.org/abs/2210.03629

---

## 🌟 背景知识（新手必读）

想象你在解决一道复杂的数学题或调查一件事情：

**纯"思考"（Chain-of-Thought）**：你在脑子里推理，步骤一、步骤二、步骤三……但如果某一步需要查资料怎么办？你无法搜索，只能依靠记忆，容易出错。

**纯"行动"**：你直接搜索、点击、获取信息，但每次行动都是盲目的，不清楚为什么要做这一步，容易走弯路。

**ReAct的思路**：把思考和行动交替进行——先想一想该做什么，然后做，看结果，再想下一步该做什么……就像一个边做边想的实习侦探：

"我需要找出XXX的答案（思考）→ 搜索XXX（行动）→ 看到结果A（观察）→ 结果A说了B，但我还需要确认C（思考）→ 再搜索C（行动）→ 现在我有足够信息了（思考）→ 最终答案是D"

这个"思考→行动→观察→思考→行动…"的循环就是ReAct。

---

## 🔬 论文做了什么？

ReAct（Reasoning + Acting）提出了一种简单有效的框架：在语言模型生成过程中，允许它交替产生"思考"步骤（自由文字推理）和"行动"步骤（调用外部工具如搜索引擎、计算器）。在HotpotQA（多跳问答）、FEVER（事实核查）、ALFWorld（游戏任务）等基准上，ReAct显著超过了纯链式思维（Chain-of-Thought）或纯行动方法，并且更具可解释性。

---

## 🛠️ 怎么做到的？

**核心思路：混合推理-行动轨迹**

传统的few-shot prompting只给模型展示（问题 → 答案）的示例。
CoT给（问题 → 思考步骤 → 答案）的示例。
ReAct给（问题 → 思考 → 行动 → 观察 → 思考 → 行动 → 观察 → ... → 答案）的示例。

**具体格式：**

```
Question: 《战争与和平》的作者在哪一年出生？

Thought 1: 我需要找到《战争与和平》的作者是谁。
Action 1: 搜索["《战争与和平》作者"]
Observation 1: 《战争与和平》是俄国作家列夫·托尔斯泰的作品。

Thought 2: 我现在知道作者是托尔斯泰，需要找到他的出生年份。
Action 2: 搜索["列夫·托尔斯泰 出生年份"]
Observation 2: 列夫·托尔斯泰（1828年9月9日—1910年11月20日）

Thought 3: 根据搜索结果，托尔斯泰生于1828年。
Action 3: Finish[1828年]
```

**工具池：**
- 搜索：SearchWikipedia（查询维基百科）
- 查询：Lookup（在页面内查找特定关键词）
- 完成：Finish（给出最终答案）

**训练方式：**

ReAct不需要额外训练！它是一种**提示工程（Prompt Engineering）**方法——只需提供几个包含"思考-行动-观察"格式的示例（few-shot examples），大型语言模型就能学会按照这种格式生成文本。

**模型：**实验主要在PaLM（540B）和GPT-3（text-davinci-002）上进行。

## 📊 主要结论

| 方法 | HotpotQA（多跳QA，EM） | FEVER（事实核查，Acc） |
|-----|---------------------|---------------------|
| Standard（无CoT，无工具） | 29.4% | 57.1% |
| CoT（链式思维，无工具） | 29.1% | 56.3% |
| Act（有工具，无思考） | 28.0% | 60.1% |
| ReAct（思考+工具） | **35.1%** | **64.6%** |
| ReAct + 少量监督 | **39.5%** | **67.3%** |

- ReAct在HotpotQA上比最佳基线提升了~6个百分点
- ReAct在FEVER上比Act（纯行动）提升了~4.5个百分点
- 人工分析显示，ReAct产生了更少的"幻觉"错误（比CoT少约65%）
- 在游戏类任务（ALFWorld）上，ReAct达到71%的成功率（vs. Act的45%）

---

## ⚠️ 局限性

1. **工具依赖**：ReAct的效果高度依赖工具的质量和可用性；如果搜索结果不准确，模型可能被误导。
2. **上下文长度限制**：思考-行动-观察的链条可能很长，超出LLM的上下文窗口限制。
3. **效率较低**：相比直接回答，ReAct需要多次工具调用，延迟更高、成本更高。
4. **错误传播**：早期推理错误可能在后续步骤中被放大，而不是被纠正。
5. **不支持并行**：ReAct是顺序执行的，无法并行调用多个工具提高效率。

---

## 💡 为什么值得关注？

ReAct是AI Agent（智能体）领域最有影响力的论文之一：

1. **现代AI Agent的基础**：ReAct的"思考-行动-观察"循环成为几乎所有现代AI Agent框架的基础设计，包括LangChain的AgentExecutor、LlamaIndex的ReAct Agent、AutoGPT等。

2. **推动工具使用范式**：ReAct推广了"LLM + 外部工具"的开发范式，使AI系统能够突破语言模型固有知识的限制，实时获取最新信息。

3. **可解释性优势**：与纯黑盒行动方法相比，思考步骤提供了AI决策过程的"思路可见性"，便于调试和用户理解。

4. **后续工作影响**：ReAct直接影响了：
   - Reflexion（结合自我反思）
   - Tree of Thoughts（多分支搜索）
   - Toolformer（工具调用的微调）
   - GPT-4 Plugins（ChatGPT插件系统）

5. **应用广泛**：ReAct已被用于构建各类AI助手：代码调试、网页浏览、数据分析、学术研究助手等，是"智能体"产品化的核心技术之一。
