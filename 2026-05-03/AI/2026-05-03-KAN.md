# KAN：用可学习函数重新定义神经网络的激活

## 基本信息

- **论文标题**：KAN: Kolmogorov-Arnold Networks
- **论文链接**：https://arxiv.org/abs/2404.19756
- **发表时间**：2024年4月
- **机构**：麻省理工学院（MIT）/ Caltech / 东北大学
- **作者**：Ziming Liu et al.（MIT IAIFI）

---

## 背景知识：神经网络是怎么工作的？

**传统神经网络（MLP，多层感知机）的结构**：

```
输入层 → [线性变换] → [固定激活函数] → [线性变换] → ... → 输出
```

关键点：
- **线性变换**：每一层把输入的数字线性组合（加权求和）
- **激活函数**：非线性函数（如 ReLU、Sigmoid、GELU），让网络可以学习复杂关系
- 激活函数是**固定的**：训练时只更新线性变换的权重，激活函数的形状不变

**数学基础——通用近似定理（Universal Approximation Theorem）**：
MLP 的理论依据：任何连续函数都可以用足够宽的神经网络来近似。

但 MLP 的这个理论来自 **Kolmogorov-Arnold 表示定理**（1957年）：
> 任何多变量连续函数都可以分解为若干个一元函数的叠加组合。

**问题**：KAN 的作者说，"我们为什么不直接按照 Kolmogorov-Arnold 定理来设计网络呢？"

---

## 这篇论文干了什么？

KAN 提出了一种**新的神经网络架构**：
- 把激活函数从"固定"变成"可学习的"
- 每条网络连接（边）都有自己的可学习激活函数
- 理论上可以更高效地近似各种函数
- 具有更好的**可解释性**：可以可视化每条边学到的函数形状

---

## 怎么干的？

### KAN 的核心思路

**MLP（传统）**：
```
节点上有激活函数，边上只有权重（数字）
y = σ(w₁x₁ + w₂x₂ + ...)
```

**KAN（新）**：
```
边上有可学习的函数，节点只做加法
y = Σ φᵢⱼ(xⱼ)
其中 φᵢⱼ 是第 j 个输入到第 i 个输出的可学习一元函数
```

**通俗理解**：
- MLP：每条连接线是一个数字（权重），节点上有一个固定形状的弯曲函数
- KAN：每条连接线是一个可以自由弯曲的函数（形状由训练决定），节点只是做加法

### 用 B-spline 实现可学习激活函数

每条边的可学习函数用 **B-spline（B样条）** 来参数化：
- B-spline 是一种通过控制点定义的平滑曲线
- 通过学习控制点的位置，就能学习任意形状的函数
- 可以增加控制点数量来提高函数的表达能力（**网格细化**）

还加入了一个**基础函数**（类似 SiLU 激活函数）加上 spline，保证训练稳定性。

### 稀疏化与可解释性

KAN 的重要特性：
1. **正则化**：用 L1 正则化鼓励大多数连接函数趋向零（稀疏化）
2. **剪枝**：删除接近零的连接，留下最关键的结构
3. **可视化**：每条边学到的函数可以画出来，直观理解网络在做什么
4. **符号回归**：可以尝试用已知数学函数（sin、exp、log 等）替换学到的函数，自动发现数学规律

## 实验结果与结论

**科学发现任务**：
- 在物理方程发现任务上，KAN 能从数据中"发现"已知的物理定律（如质量守恒方程）
- 在数学特殊函数（如 Bessel 函数）的近似上，比 MLP 精度更高、参数更少

**回归任务**：
- 对于**光滑函数**的近似，相同精度下 KAN 所需参数量远少于 MLP（因为激活函数本身可以携带信息）
- 在低维但高度结构化的问题（如物理模拟）上表现优秀

**可解释性示例**：
- 训练 KAN 拟合混沌物理方程，剪枝+符号化后，得到的网络结构直接对应已知物理公式

**关键结论**：KAN 提供了比 MLP 更有表达力（单位参数量）和更好可解释性的替代方案，特别适合需要从数据中发现数学规律的科学计算场景。

---

## 局限性（Limitations）

1. **训练速度慢**：目前实现中，KAN 的训练速度比同等规模 MLP 慢约 **10 倍**，在大规模任务上不实用

2. **超参数敏感**：网格点数量、spline 阶数等超参数对性能影响大，调参困难

3. **大规模任务性能未验证**：论文中的实验规模很小（几十到几百万参数），对 LLM 级别（数十亿参数）的任务性能未知

4. **不适合所有任务**：在标准的图像分类、语言建模等任务上，并不明显优于优化过的 MLP

5. **理论与实践差距**：理论上优于 MLP，但实际工程中的挑战（如 spline 的梯度计算、GPU 并行化）限制了应用
