# DeepSeekMath + GRPO：用强化学习让 LLM 学会真正的数学推理

## 基本信息

- **论文标题**：DeepSeekMath: Pushing the Limits of Mathematical Reasoning in Open Language Models
- **论文链接**：https://arxiv.org/abs/2402.03300
- **发表时间**：2024年2月
- **机构**：DeepSeek AI（深度求索，中国）

---

## 背景知识：为什么数学对 LLM 这么难？

**数学推理的特殊性**：
- 数学问题只有**对或错**，没有"差不多对"——这和写文章、聊天很不一样
- 数学需要**多步骤推理**：每一步的错误会累积，最终答案错误
- 需要**精确计算**：语言模型对大数计算（比如 997 × 853）容易出错
- 数学有**严格逻辑**：需要明确理解"因为...所以...""若...则..."

**当时的困境（2024年初）**：
- GPT-4 在 MATH 基准（高中竞赛题）上约 42-72%（取决于测试方式）
- 开源模型（如 Llama 2）在 MATH 上只有 10-20%
- 数学是 LLM 公认的"软肋"之一

**为什么强化学习适合数学？**
强化学习（RL）的关键要素：
1. **明确的奖励信号**：答案对 → +1，答案错 → 0。数学有天然的对错判断！
2. **探索与利用**：模型可以尝试不同的解题思路，被奖励引导走向正确路径

---

## 这篇论文干了什么？

DeepSeekMath 做了两件事：
1. **数据层面**：从互联网上精心筛选高质量数学数据，构建 **1200 亿 token** 的数学预训练数据集
2. **训练方法层面**：提出 **GRPO（Group Relative Policy Optimization）**，一种比 PPO 更简单高效的强化学习算法，专门为 LLM 数学推理设计

最终：开源 **DeepSeekMath-7B** 模型，在 MATH 基准上达到 **51.7%**，超越所有 7B-34B 级别的开源数学模型，接近 GPT-4 水平。

---

## 怎么干的？

### 第一步：数学数据工程

**筛选过程**：
1. 从 Common Crawl（全网爬取数据）中筛选数学相关内容
2. 使用 **fastText 分类器** 把数学内容从普通网页中识别出来
3. 迭代式数据过滤：用已有数学内容训练分类器 → 用分类器找更多数学内容 → 反复迭代
4. 去重、质量过滤

最终获得 **1200 亿 token** 的数学数据（覆盖：数学论文、解题过程、数学论坛如 Math StackExchange、代码中的数学实现等）

### 第二步：分阶段训练

**阶段一：继续预训练**
- 在 DeepSeek-Coder-Base-7B 上继续预训练
- 数据：数学数据 + 代码数据 + 自然语言数据（混合比例经过实验调优）
- 代码数学一起训练的原因：代码训练增强了精确步骤执行的能力

**阶段二：监督微调（SFT）**
- 用高质量的数学解题示例微调
- 同时训练两种解题风格：
  - **思维链（CoT）**：逐步推理过程（文字版）
  - **代码解题（PoT, Program of Thought）**：写 Python 代码求解

**阶段三：GRPO 强化学习**（核心创新）

### GRPO：群体相对策略优化

传统 RLHF（PPO）需要：
- 一个奖励模型（Reward Model）来评分
- 一个价值模型（Critic/Value Model）来估计未来期望奖励
- 训练两个模型，成本高、不稳定

**GRPO 的创新**：不需要单独的价值模型！

**GRPO 的做法**：
1. 对同一道数学题，让模型生成 **G 个** 不同的解答（如 G=8 个）
2. 判断每个解答是否正确（对错二元判断）
3. 计算相对奖励：
   - 组内正确解答比例高 → 该解答的相对奖励较低（因为容易）
   - 组内正确解答比例低 → 正确解答的相对奖励高（因为难得）
   
```
相对奖励 = (当前解答奖励 - 组内平均奖励) / 组内奖励标准差
```

4. 用这个相对奖励更新模型，鼓励模型多产生正确解答

**直觉理解**：
- 如果一道题模型 8 次中 8 次都答对了 → 这道题太简单，不需要强化
- 如果 8 次中只有 1 次答对 → 这道题有难度，正确的那个解题思路特别值得强化
- 如果 8 次中 4 次答对 → 优先强化这 4 个正确解题路径

## 实验结果与结论

在 MATH（高中竞赛数学）基准上：

| 模型 | MATH 分数 |
|------|----------|
| GPT-4（2023）| ~42% |
| Llama 2 70B | 13.5% |
| **DeepSeekMath-7B-RL** | **51.7%** |
| Gemini Ultra（2024）| 53.2% |

**关键发现**：
- GRPO 比 PPO 更稳定，训练成本更低（不需要单独的价值模型）
- 代码和数学联合训练比纯数学训练效果更好
- RL 阶段（GRPO）相比仅 SFT，MATH 提升约 **7-10 个百分点**

**关键结论**：通过大规模高质量数学数据预训练 + GRPO 强化学习，7B 的小模型可以在数学推理上接近最顶级的大模型（GPT-4）；GRPO 作为一种简化的 RL 算法，为 LLM 强化学习提供了新的高效路径。

---

## 局限性（Limitations）

1. **奖励稀疏问题**：对于极难的题目（8 次全部答错），GRPO 没有任何正确样本可以学习，无法提升

2. **只用对错二元奖励**：无法给"解题思路对但计算错误"的答案部分奖励

3. **过拟合数学风格**：强化学习后，模型的非数学能力（写作、常识推理）可能有所退化

4. **数学竞赛以外的数学任务**：对需要创意和估算的"应用数学"场景效果未充分验证

5. **数学数据收集成本**：筛选 1200 亿 token 高质量数学数据的工程成本依然很高

## 为什么值得读？

DeepSeekMath 和 GRPO 是 **"强化学习 + 数学推理"** 这一重要研究方向的奠基性工作。后来的 DeepSeek-R1 和各种"慢思考"模型都延续了这条技术路线。如果你想理解 LLM 是如何通过 RL 学会推理的，这是一篇入门必读论文。
