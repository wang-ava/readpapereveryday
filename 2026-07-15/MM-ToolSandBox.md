# MM-ToolSandBox: A Unified Framework for Evaluating Visual Tool-Calling Agents

## Spotlight（今日速读）
这是一套从“真实交互流”而非单点问答任务评估模型能力的基准：视觉工具调用 agent 在多轮、状态变更和工具错序场景下依然崩得很厉害。论文为该能力给出了可量化测试面板，也告诉我们当前模型的短板更多在“看清”而非“会不会规划”。

## 核心问题
现有 tool-calling 评测多偏向文本指令静态判断，忽略了视觉输入、状态变化和交互纠错。论文核心问题是：如何在多轮、视觉驱动、可变状态任务中统一评估 agent 工具执行能力？

## 方法概要
- 构建覆盖 16 个应用域、500+ 可调用工具的状态化执行环境。
- 通过可流式生成的场景流程设计多图像与多轮对话任务，包含目标变更、纠错、状态冲突等行为。
- 引入 258 个人工验真场景和 50 个交互型变体，形成可追踪失败类别的评测池。
- 对 12 个 SOTA model 做统一任务分解、工具调用、错误归因分析。

## 主要贡献
- 首次提出支持高交互密度的视觉工具调用统一评测框架。
- 将“视觉精度不足”与“规划不足”区分为不同失败机制。
- 建立工具调用失败的可解释 error taxonomy，便于定位模型短板。
- 开源 benchmark 和执行器，便于下游模型评测与复现实验。

## 关键实验或结果
- 最强模型在全部基准上成功率仍低于 50%，显示视觉工具调用仍处于早期阶段。
- 约 53% 失败来自图像信息提取错误，而不是任务规划失误。
- 小模型更常见“不知道怎么做”，大模型更常见“看错图像细节”，呈现不同规模瓶颈。

## 适合关注的原因
对 Agentic 产品来说，问题不再是“会不会调用 API”，而是“在视觉干扰和上下文漂移下能否持续完成任务”。该基准可直接用于筛选可上线的 agent 版本。

## 局限性或待验证点
- 工具场景仍以公开生成体系为主，真实业务系统的安全策略和权限模型未全覆盖。
- 成功率上界与任务分布依赖场景生成器，可能低估某些行业闭环任务的复杂度。
- 论文未给出细粒度跨地区语言与界面风格鲁棒性评估。

## 对后续研究/应用的启发
- 可作为 agent 迭代的回归测试：每个模型版本都用固定种子场景进行失败谱监控。
- 建议把“视觉感知失败率”与“工具链错误率”分离建模，制定针对性训练预算。
- 对企业可行路径：先在该类 benchmark 对齐视觉解析，再引入业务 domain-specific 工具策略。

## 一句话总结
MM-ToolSandBox 把视觉工具调用从“单点能力展示”拉回到“稳定执行系统流程”，并把当前瓶颈明确指向视觉理解与状态鲁棒性。

## 标准化研究框架
- **Research question：** 面对真实交互环境，视觉工具调用 agent 的可用性瓶颈究竟来自规划失败还是感知失败，如何可量化区分？  
- **Literature：** 现有 benchmark 常集中在纯文本 tool-use 或单轮任务，本文补齐多轮视觉状态任务这一缺失。  
- **Theory：** 在非社会科学语境下可等同于“感知-规划-执行闭环”在 noisy 环境下的可靠性建模，关注失败因果归因可解释性。  
- **Hypotheses：** H1：统一 benchmark 可揭示模型规模上升带来的错误结构转移；H2：视觉提取错误是主要瓶颈；H3：状态建模能力显著影响工具调用成功率。  
- **Method：** 构建 500+工具、16域、多轮任务集，进行跨模型、跨场景统一评测与错误归因。  
- **Data and Analysis：** 以 258+50 场景为核心数据，按失败类型（认知、提取、状态）做分层统计分析。  
- **Findings：** 报告显示 best model 成功率 <50%，且感知失败在总体错误中占比高。  
- **Conclusion：** 未来提升应优先投入视觉信息提取与状态追踪，而非单一扩大模型规模。
