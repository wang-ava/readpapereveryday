## 背景知识：开源 LLM 的意义

### 闭源 vs. 开源

2023年，最强的 LLM 几乎都是闭源的：
- GPT-4（OpenAI）：只能通过 API 调用，无法查看权重
- Claude（Anthropic）：同样仅 API
- Gemini（Google）：同样仅 API

**闭源的问题**：
1. 使用成本高：按 token 收费
2. 数据隐私：你的数据会发给第三方服务器
3. 研究受限：无法研究模型内部机制
4. 部署不灵活：不能在本地或私有云部署

**开源模型**（公开权重）让研究者和企业可以：
- 本地运行，数据不离开服务器（医疗、金融等敏感场景）
- 自由微调，适应特定领域
- 研究模型内部，推进 AI 安全研究
- 降低 AI 使用门槛

### LLaMA 1 的局限

2023年2月，Meta 发布了 **LLaMA 1**（7B, 13B, 33B, 65B参数）。

**优点**：相对小，性能强（65B 比 GPT-3 175B 还要好！）  
**缺点**：**研究许可证，不允许商业使用**。很多公司想用，但法律上不允许。

---

## 这篇论文干了什么？

**LLaMA 2** 是 Meta 2023年7月发布的：

1. **完全开源，允许商业使用**（每月活跃用户不超过7亿）
2. **更多训练数据**：2万亿 tokens（vs. LLaMA 1 的 1.4万亿）
3. **更长上下文**：4096 tokens（vs. LLaMA 1 的 2048）
4. **LLaMA 2-Chat**：经过 RLHF 对齐的聊天版本，堪比 ChatGPT
5. **详细的安全报告**：比以往任何开源模型都更透明

---

## 怎么做的？

### 预训练：更多更好的数据

LLaMA 2 的预训练数据：

| 来源 | 描述 |
|------|------|
| 公开网络文本 | 过滤后的 Common Crawl |
| 学术数据 | ArXiv、StackExchange 等 |
| 代码 | 开源代码 |
| 书籍 | 电子书 |
| 维基百科 | 多语言维基 |

**与 LLaMA 1 相比**：增加了约40%的数据量，更多地过滤掉低质量内容。

**训练规模**：
- 7B 参数：1万亿 tokens
- 13B 参数：1.3万亿 tokens
- 70B 参数：2万亿 tokens（最大，最优）

### 模型结构改进

| 改进点 | LLaMA 1 | LLaMA 2 |
|--------|---------|---------|
| 上下文长度 | 2048 tokens | **4096 tokens** |
| 分组查询注意力（GQA）| 无（只有70B用）| 34B、70B 使用 |
| Pre-normalization | 是 | 是 |
| SwiGLU 激活 | 是 | 是 |

**分组查询注意力（GQA）**：在大模型（34B、70B）上使用，减少推理时的 KV Cache 大小，加快推理速度。

### LLaMA 2-Chat：打造类 ChatGPT 的聊天模型

Meta 额外投入了大量工作，对 LLaMA 2 进行对齐：

**Stage 1：监督微调（SFT）**
- 收集了 27,540 条高质量的人工对话数据
- 格式：用户问题 + 理想的助手回答
- 强调质量而非数量（精挑细选，不是大量平庸数据）

**Stage 2：RLHF（人类反馈强化学习）**

与 InstructGPT 类似，但有几个改进：

**两个独立的奖励模型**：
- 帮助性奖励模型（Helpfulness RM）：判断回答是否有用
- 安全性奖励模型（Safety RM）：判断回答是否无害

**原因**：安全和帮助有时是矛盾的（"帮助"用户做危险的事 vs. 拒绝并保持安全），用两个模型分别处理更好。

**Ghost Attention（GAtt）**

长对话中的一个问题：在很长的多轮对话后，模型会"忘记"最初的系统指令（如"你是医疗助手，只回答医学问题"）。

GAtt 的解决方案：训练时，把系统提示"幽灵般地"附加到对话中每一条消息后面，让模型在整个对话过程中始终"记住"系统指令。

**迭代式 RLHF**

不是训练一次就结束，而是多次迭代：
1. 训练奖励模型
2. 用奖励模型做PPO
3. 用改进后的模型收集新数据
4. 重新训练奖励模型
5. 循环...

每次迭代，收集更难的例子，持续提升模型能力。

### 安全措施

LLaMA 2 的安全工作比以往任何开源模型都更系统：

**1. 安全数据**：额外收集了专门的安全对话数据，覆盖各种有害场景。

**2. Red Teaming（红队测试）**：组织内部团队像"攻击者"一样测试模型，找出可以被滥用的场景。

**3. 透明报告**：论文详细报告了各种安全测试的结果（有害回复率等）。

## 主要结论

### 性能对比

| 模型 | MMLU | BBH | HumanEval |
|------|------|-----|-----------|
| GPT-3.5 | 70.0% | 58.1% | 48.1% |
| **LLaMA 2-70B** | **68.9%** | **51.2%** | **29.9%** |
| Falcon-40B | 55.4% | 37.1% | - |

LLaMA 2-70B 接近 GPT-3.5，大幅超越其他开源模型。

### LLaMA 2-Chat 对比

在人类偏好评测中：
- LLaMA 2-Chat-70B vs. ChatGPT（GPT-3.5）：35.8% 偏好 LLaMA，41.8% 偏好 ChatGPT（差距不大）
- LLaMA 2-Chat 安全性：超越了大多数开源聊天模型

---

## 局限性

1. **知识截止日期 2022年12月**：不了解此后的事件（如 GPT-4、Claude 3 等）
2. **与 GPT-4 差距仍大**：对比 GPT-4，在推理、长文本理解等方面差距明显
3. **代码能力有限**：相比 GPT-4 Code Interpreter 或 Claude 3 在代码上差距大
4. **上下文窗口短**：4096 tokens，而 Claude 3 可以 200K tokens
5. **34B 版本未发布**：论文提到了 34B，但实际只发布了 7B、13B、70B
6. **商业限制**：月活用户超过7亿需要额外授权（这排除了大型平台如 Google、Microsoft 直接使用）

## 为什么值得关注？

LLaMA 2 是 **2023年下半年开源 AI 生态的核心**：

1. **生态效应**：LLaMA 2 发布后，涌现出大量基于它的微调模型：
   - Vicuna（通用聊天）
   - WizardLM（复杂指令）
   - CodeLlama（代码）
   - Chinese-LLaMA-2（中文）
   - 医疗、法律、科学等各领域专用模型

2. **推动了开源 AI 社区**：
   - HuggingFace 上的模型数量激增
   - 普通研究者可以在消费级 GPU 上运行大模型

3. **商业可用**：终于有一个可以合法商用的强大开源模型

**历史地位**：LLaMA 2 是开源 LLM 发展的关键节点，它标志着开源社区真正能与闭源商业模型正面竞争的时代开始。
