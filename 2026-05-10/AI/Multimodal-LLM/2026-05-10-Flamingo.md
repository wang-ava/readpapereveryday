# 2026-05-10 | Flamingo: 面向少样本学习的视觉语言模型

> **论文链接**: https://arxiv.org/abs/2204.14198  
> **发表**: NeurIPS 2022  
> **机构**: DeepMind  
> **作者**: Jean-Baptiste Alayrac, Jeff Donahue, Pauline Luc 等  
> **领域**: 多模态大语言模型 / 少样本视觉问答

---

## 背景知识：语言模型的少样本能力是什么？

### GPT-3 的"上下文学习"

2020年 GPT-3 展示了一个惊人的能力——**少样本学习（Few-Shot Learning）**：

你不需要重新训练模型，只需要在输入里提供几个例子，GPT-3 就能"举一反三"：

```
输入：
将以下句子翻译成法语：
英文：The cat is on the mat → 法语：Le chat est sur le tapis
英文：The dog is in the car → 法语：Le chien est dans la voiture
英文：The bird is in the tree → 法语：[GPT-3填写答案]

GPT-3自动完成：L'oiseau est dans l'arbre
```

这是"上下文学习"（In-Context Learning）——模型根据上下文中的例子，推断出任务规则，无需任何训练。

**问题**：GPT-3 是纯文字模型，这种能力能否扩展到图像？

### 传统视觉语言模型的痛点

在 Flamingo 之前，多模态 AI（能同时处理图像和文字的模型）都需要：
1. 大量图像标注数据
2. 针对每个新任务重新微调
3. 为每种任务设计专门的模型结构

能不能有一个模型，像 GPT-3 那样，只给几个图文例子，就能完成新的视觉任务？

---

## 这篇论文干了什么？

**Flamingo** 是第一个在视觉任务上成功展示 GPT-3 式少样本学习能力的模型：

1. **交错图文序列**：能处理"图1，问题，图2，问题，新图，新问题"这样的序列
2. **少样本视觉问答**：只给16个图文例子，就能在新图像上做问答
3. **冻结预训练组件**：高效复用现有大模型（不需要从头训练）
4. **16-shot 超过有监督 SOTA**：在6/16个任务上，仅16个例子的Flamingo超越了需要大量标注的专用模型

---

## 怎么做的？

### 核心架构：冻结 + 新增交叉注意力

Flamingo 最重要的设计思想：**不改变现有大模型，而是在它们之间架桥**。

```
视觉模型（NFNet，冻结）     语言模型（Chinchilla 70B，冻结）
      ↓                              ↓
  图像特征                      文字特征
      ↓                              ↓
  Perceiver重采样器           ← ← 交叉注意力层（新增，可训练）
  （把大量图像token            （让语言模型"看"图像特征）
   压缩成少量固定token）
```

**为什么冻结现有模型？**
- NFNet（图像模型）：已经在海量图像上预训练，代价极高
- Chinchilla（语言模型）：已经学会了大量语言知识，代价极高
- 只需要训练连接它们的"桥"，成本低得多

**新增的关键模块：Perceiver Resampler**

图像编码后会有大量 token（比如一张图可能有 10,000 个像素块 token），但语言模型不能处理这么多。Perceiver Resampler 用64个"学习的查询向量"将大量图像 token 压缩成固定的64个 token，大幅减少计算量。

**交叉注意力层（Cross-Attention Layers）**

在语言模型的 Transformer 层之间，插入新的交叉注意力层：
- 语言模型在生成文字时，可以"查询"图像特征
- 类比：语言模型在阅读时，随时可以"抬头看一眼图像"

### 训练数据：多种多样的大规模数据

**数据1：ALIGN（图文对，43亿条）**
- 从网络爬取的图文对（对齐图像）
- 主要提供视觉-语言对齐基础

**数据2：LTIP（长文本图文交错，3.82亿文档）**
- 包含多张图像穿插在长文本中的网页
- 关键：让模型学会处理"交错图文序列"

```
示例文档：
[图1：一碗意大利面]
今天我尝试做了一道意大利面食谱...
[图2：加调料的过程]
然后我加入了大量的橄榄油...
[图3：成品]
最终成品看起来还不错！
```

**数据3：VTP（视频图文对，2.7亿）**
- 视频帧 + 字幕
- 进一步丰富视觉-语言关系

### 少样本推理：交错输入格式

训练完成后，Flamingo 可以接受这样的输入：

```
[图：一只橙色的猫]
问题：这只猫是什么颜色？
答案：橙色

[图：一只黑色的狗]
问题：这只动物是什么？
答案：狗

[图：一只白色的鸟]
问题：这只鸟可能在哪里生活？
答案：[Flamingo生成]
```

通过前两个例子，模型"理解"了任务的格式和类型，然后对新图像作答。

## 主要结论

### 在16个视觉语言任务上的评测

**少样本设置（仅 4-32 个例子）**：

| 类别 | 任务数 | Flamingo超过有监督SOTA |
|------|--------|---------------------|
| 视觉问答（VQA）| 5 | 3/5 |
| 图像描述 | 4 | 2/4 |
| 视觉推理 | 4 | 1/4 |
| 光学字符识别（OCR）| 3 | 0/3 |

**总计**：16个任务中，**6个任务**的 Flamingo 16-shot 超越了需要大量标注数据的有监督 SOTA！

### 模型规模的影响

Flamingo 有三个版本：3B、9B、80B（参数量指语言模型部分）

- 随着规模增大，性能持续提升（符合 Scaling Laws）
- 80B 版本效果最好

---

## 局限性

1. **依赖封闭的预训练组件**：NFNet 和 Chinchilla 都是 DeepMind 内部模型，论文发表时未开源，其他研究者无法直接复现
2. **训练成本极高**：尽管冻结了主要模型，交叉注意力层的训练仍需要大规模计算
3. **在标准任务上不如专用模型**：对于有大量标注数据的任务（如 VQA v2），有监督专用模型仍然更强
4. **没有处理 3D/视频的完整能力**：尽管用了视频训练数据，对视频的理解仍然有限
5. **语言模型偏差**：语言模型（Chinchilla）的偏见和局限会影响多模态输出

---

## 为什么值得关注？

Flamingo 是现代多模态 AI（GPT-4V、Gemini、LLaVA 等）的**直接先驱**：

1. **架构影响深远**：冻结视觉编码器 + 冻结语言模型 + 可训练桥接层，这个范式被大量后续工作采用
2. **证明了多模态少样本学习的可行性**：在视觉任务上也能做 GPT-3 式的上下文学习
3. **DeepMind 技术展示**：证明了 DeepMind 在多模态AI领域的技术实力

**历史脉络**：
```
Flamingo（2022）→ BLIP-2（2023）→ LLaVA（2023）→ GPT-4V（2023）→ Gemini（2023）
```

所有这些后续工作都在某种程度上受到 Flamingo 架构思想的影响。
