# PROJECTMEM: A Local-First, Event-Sourced Memory and Judgment Layer for AI Coding Agents

轻量级且可追溯的事件日志让 AI 编码代理从“会忘记”变成“会继承”：它把每次会话的决策、尝试和失败结构化记录，能在下次调用时自动提醒重复错误。对想把代码助手长期纳入工程流程的人，这是“把记忆和治理一起做进代理”的直接示例。

- **核心问题**：AI 编码代理在长会话中容易遗失上下文和决策依据，导致重复试错、修复回归、以及缺乏可追溯记录，严重影响开发效率与稳定性。
- **方法概要**：论文提出 projectmem 的本地优先（local-first）架构：
  1. 将开发过程建模为 append-only 的纯文本事件流（问题、尝试、修复、决策、备注）；
  2. 将事件流按确定性规则压缩为结构化摘要；
  3. 通过 MCP 向代理提供工作记忆；
  4. 加入前置判定（pre-action gate）阻止重复危险操作（例如重新改动高风险文件或复现已知失败模式）；
  5. 全流程离线运行，不依赖遥测。
- **主要贡献**：
  1. 把事件溯源（event sourcing）引入代理“状态记忆”与“行为审计”；
  2. 提出 Memory-as-Governance 设计：记忆不仅检索历史，而是直接干预下一步行为；
  3. 通过 14 MCP tools、19 CLI 命令和 37 自动化测试形成可直接接入工具链的工程化实现；
  4. 在 10 个项目、207 个记录事件的两个月自学习中展示其工程可行性。
- **关键实验或结果**：实验侧重于工程验证，核心结论是：通过结构化事件记录和判定钩子，可减少上下文重建成本与重复修复；论文给出的实现细节显示其在中等规模开发任务中具备可复现的效率收益。
- **适合关注的原因**：
  1. 从“生成能力”转向“持续执行能力”；
  2. 与 MCP/agent 工具链天然兼容；
  3. 对 CI、日志与治理要求高的团队，能直接降低隐式状态带来的风险。
- **局限性或待验证点**：
  1. 当前评测主要集中在编码场景，不一定泛化到医疗、法务等高规范领域；
  2. 缺少与主流闭源 coding agent 的大规模 A/B 比较；
  3. 事件写入成本与上下文压缩质量对长期运行仍需测评。
- **对后续研究/应用的启发**：可沿两条线延展：其一，将事件模型扩展到跨项目共享知识图谱，实现团队级代理知识库；其二，将 pre-action gate 与政策合规规则耦合（如安全规范、审计阈值、权限边界）。
- **Obsidian 快速浏览一句总结**：把 coding agent 从“每次重启重学”改造成“有记忆、有判断、有审计链路”的生产级协作单元。

## 标准化研究框架
- **Research question：** 如何通过本地化、可追溯的事件日志机制，提升 LLM coding agent 在长期软件开发任务中的稳定性、可复用性与可审计性？
- **Literature：** 与检索增强、tool use 代理和基于记忆的代理框架相关，但相比传统会话缓存，它强调事件溯源、离线优先和行为前置拦截。
- **Theory：** 本文更偏工程机制：以事件溯源保证历史不丢失，以确定性摘要保证可供模型高效读取，并通过规则网关将行为风险外化为可干预信号。
- **Hypotheses：** 若历史以最小语义单元持久存储并被可执行摘要化，代理会减少重复错误；并且若加入“行动前判定”，可提高系统稳定性与安全性。
- **Method：** 构建 append-only 事件日志 + MCP 摘要服务 + 前置判定器；在多个项目内进行滚动自学习并测量工程指标。
- **Data and Analysis：** 采用 10 个项目、207 条 logged events 的两个月实验；分析集中在任务成功率、重复失败率下降趋势与上下文复用效率。
- **Findings：** 证据显示在工程流程中引入“结构化记忆+行为门控”可缓解上下文漂移与重复调试问题。
- **Conclusion：** 本文为“代理记忆”提供了可执行系统级范式，但规模化部署前仍需跨域泛化与安全规则的一致化验证。
