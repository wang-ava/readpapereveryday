# 20260422-DiT：用 Transformer 替换 U-Net 的可扩展扩散模型

**论文链接：** https://arxiv.org/abs/2212.09748

**发表信息：** ICCV 2023 | Peebles & Xie（UC Berkeley）

---

## 背景知识：扩散模型中的 U-Net 是什么？

在前面介绍的 **LDM/Stable Diffusion** 中，扩散过程的核心是一个"去噪网络"——它负责从噪声中逐步还原出图像。在 Stable Diffusion 中，这个网络采用的是 **U-Net 架构**：

**U-Net** 是一种特殊的卷积神经网络（CNN），它有个形似字母 "U" 的结构：
- **下采样（编码）路径**：把图像逐步压缩，提取抽象特征
- **上采样（解码）路径**：逐步还原图像分辨率
- **跳跃连接**：连接编解码对应层，保留细节

U-Net 在医学图像分割（2015年）中诞生，后来被广泛用于图像生成。

**但 U-Net 有个问题**：CNN 架构的**可扩展性（Scalability）不如 Transformer**——加大模型规模时，U-Net 的性能提升没有 Transformer 那么稳定和可预测。

---

## 这篇论文做了什么？

**DiT**（**Di**ffusion **T**ransformer）用 **Vision Transformer（ViT）替换了扩散模型中的 U-Net**，得到了一个更简洁、更可扩展的图像生成模型。

核心发现：**扩散模型的性能与 Transformer 的规模（参数量 + 计算量）呈强正相关**，遵循 NLP 中已知的缩放定律（Scaling Laws）。

---

## 怎么做到的？

### 第一步：把图像变成 Token 序列
- 用 LDM 的 VAE 编码器把图像压缩到潜在空间（如 32×32×4 的张量）
- 把潜在表示切成小块（Patch），展平成序列（类似 ViT）
- 每个 Patch 就是一个 Token，整图变成 256 个 Token 的序列

### 第二步：Transformer 处理序列
- 用标准的 Transformer 块处理这个 Token 序列
- 每个块：Layer Norm → Self-Attention → Layer Norm → MLP
- 迭代 T 次去噪步骤

### 关键设计：条件注入方式
如何把去噪步骤 t（时间步）和类别条件 c 注入 Transformer？论文对比了四种方式：
1. 直接加到 Token 序列里（In-context conditioning）
2. 用交叉注意力注入
3. 用 AdaLN（自适应层归一化）——最终选择这个，效果最好
4. 用 AdaLN-Zero（零初始化的 AdaLN）

### 模型家族
DiT-S、DiT-B、DiT-L、DiT-XL（参数从 33M 到 675M），系统验证缩放规律

---

## 结论

1. **扩散模型也服从缩放定律**：模型越大、计算量越多（GFLOPs），生成图像的 FID 分数（越低越好）越好，关系非常规律
2. **超越 U-Net 基线**：DiT-XL/2 在 ImageNet 256×256 生成上，FID = 2.27，超越了此前所有 LDM 模型
3. **架构更简洁统一**：把图像生成的骨干网络统一到 Transformer，和 NLP、视觉理解等领域共用一套架构
4. **为 Sora 铺路**：OpenAI 的视频生成模型 Sora 采用了 DiT 架构（视频 Transformer），DiT 成为下一代生成模型的基础
5. **商业影响巨大**：Stability AI 的 Stable Diffusion 3.0、Black Forest Labs 的 FLUX 都采用了 DiT 架构

---

## 局限性

1. **训练计算量更大**：DiT 在相同参数量下，比 U-Net 需要更多 FLOPs（计算量）来达到相同的性能，推理成本更高
2. **二次复杂度**：标准 Transformer 的自注意力仍是 O(n²)，对于高分辨率图像（更多 Patch）计算量急剧增加
3. **U-Net 的感应偏置（Inductive Bias）**：U-Net 的多尺度特征和跳跃连接是专门为图像设计的，DiT 放弃了这些，某些局部细节生成可能不如 U-Net
4. **视频生成仍有挑战**：虽然 DiT 可以扩展到视频（如 Sora），但视频的时序一致性、长视频生成等问题尚未完全解决
5. **开源版本有限**：DiT 本身开源，但基于 DiT 的商业产品（Sora 等）核心改进没有公开
