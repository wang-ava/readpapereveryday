# GPT-3：只需几个例子就能学会新任务——少样本学习的奇迹

## 基本信息

- **论文标题**：Language Models are Few-Shot Learners
- **作者**：Tom Brown 等，OpenAI
- **发表会议**：NeurIPS 2020
- **论文链接**：https://arxiv.org/abs/2005.14165

---

## 背景知识：从 GPT-1 到 GPT-3 的演进

### GPT 系列的核心思想

GPT（Generative Pre-trained Transformer）是 OpenAI 开发的一系列生成式语言模型：

**语言模型的训练目标**：给定前面的词，预测下一个词。
> "今天天气___" → 模型预测 "很好"

这个看似简单的任务，如果在足够多的文本数据上训练，会迫使模型学习：
- 语法（什么词跟什么词搭配）
- 语义（概念和关系）
- 世界知识（历史、科学、常识）
- 推理能力（如果A则B类型的推理）

**从 GPT-1（2018，1.17亿参数）到 GPT-2（2019，15亿参数）再到 GPT-3（2020，1750亿参数）**，OpenAI 的核心发现是：**规模越大，涌现能力越强**。

### 什么是少样本学习（Few-Shot Learning）？

传统机器学习：给10000个例子学会做某件事  
少样本学习：给3个例子就能学会做某件事  
零样本学习：一个例子都不给，直接做

GPT-3 论文最惊人的发现：**超大语言模型不需要微调，只需要在提示词里给几个例子，就能完成各种新任务**。

这被称为 **In-Context Learning（上下文学习）**。

---

## 这篇论文干了什么？

GPT-3 训练了一个 **1750亿参数**的自回归语言模型（比 GPT-2 大100倍），发现：

1. 规模带来涌现：在足够大的规模下，模型展现出很多在小模型上看不到的能力
2. 真正实现了**零样本**（Zero-shot）、**单样本**（One-shot）、**少样本**（Few-shot）学习，不需要任何梯度更新，只靠提示词中的示例
3. 在翻译、问答、填空、推理等几十个基准上都取得了强劲性能

---

## 怎么做到的？

### 模型架构

GPT-3 使用的是标准 **Transformer 解码器**（只有解码器，没有编码器，因为语言生成只需要从左到右读取序列）。

核心参数：
- **1750亿参数**（vs GPT-2: 15亿，BERT-large: 3.4亿）
- 96层 Transformer
- 嵌入维度12288
- 96个注意力头
- 最大上下文长度2048个 token

### 训练数据

- **CommonCrawl**（清洗过的互联网爬取文本，质量过滤后约570GB）：权重最大
- **WebText2**（Reddit 高赞链接的文章）
- **Books1, Books2**（电子书）
- **维基百科**

总训练 token 数：约 **3000亿个 token**

### In-Context Learning：不修改参数就能学习

GPT-3 最重要的能力：**In-Context Learning（上下文学习）**

将"任务描述 + 几个例子 + 问题"全部写在提示词里，模型直接输出答案：

**Few-shot 示例**（英译法）：
```
sea otter => loutre de mer
peppermint => menthe poivrée
plush girafe => girafe peluche
cheese =>
```
模型直接输出："fromage"（法语的"奶酪"）

模型没有真正"学习"这个任务（参数没有更新），而是利用预训练时学到的模式匹配能力，识别出"这是翻译任务"并生成答案。

**三种模式对比**：
| 模式 | 示例数量 | 是否微调参数 |
|------|---------|-------------|
| Zero-shot | 0个（只有任务描述） | 否 |
| One-shot | 1个 | 否 |
| Few-shot | 几十个（受上下文长度限制） | 否 |

### 规模的力量：Scaling Laws

OpenAI 在 GPT-3 之前的研究（Scaling Laws for Neural Language Models，2020）发现：

> 语言模型的性能与参数量、数据量、计算量之间存在**幂律关系**——三者都需要同步增加，才能最有效地提升性能。

GPT-3 是这一规律的验证：1750亿参数 + 3000亿 token + 大量算力 → 涌现出了小模型没有的能力。

---

## 结论

1. GPT-3 在**少样本设置**下，在很多任务上性能接近或超越经过微调的 BERT 类模型
2. 在 SuperGLUE（NLP理解基准）上的少样本性能非常强
3. GPT-3 能写代码、写作文、做算术、回答常识问题……尽管不总是正确
4. 证明了模型规模是提升能力的有效路径，开启了后来 GPT-4、Claude、Gemini 等大模型的时代
5. In-Context Learning 成为大模型应用的核心范式

---

## 局限性

1. **推理能力有限**：需要多步推理的复杂问题（如数学题）经常出错（后来 Chain-of-Thought 提示解决了部分问题）
2. **幻觉严重**：可能自信地给出错误事实、编造文献，在知识密集型任务上不可靠
3. **没有对话能力**：GPT-3 只会"续写"，没有对话和指令遵循能力（ChatGPT 后来通过 RLHF 解决）
4. **极高的训练成本**：估计训练一次 GPT-3 花费 400-1200万美元，只有极少数组织能训练
5. **伦理和安全问题**：可能生成有害内容、偏见内容、虚假信息，且规模越大风险越大
6. **无法更新知识**：知识截止于训练数据，无法获取最新信息（无实时搜索能力）

---

## 一句话总结

GPT-3 训练了1750亿参数的超大语言模型，发现超大规模带来了"少样本上下文学习"的涌现能力——只需在提示词里给几个例子，无需任何参数更新就能完成全新任务，证明了"更大就是更强"的 Scaling Law，直接启发了后续 ChatGPT、GPT-4 的诞生。
