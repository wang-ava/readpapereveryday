# LoRA：用1%的参数微调大模型，效果媲美全量训练

## 基本信息

- **论文标题**：LoRA: Low-Rank Adaptation of Large Language Models
- **作者**：Edward Hu 等，微软
- **发表会议**：ICLR 2022
- **论文链接**：https://arxiv.org/abs/2106.09685
- **代码**：https://github.com/microsoft/LoRA

---

## 背景知识：大模型微调的挑战

### 全量微调（Full Fine-Tuning）是什么？

预训练大模型（如 GPT-3）虽然通用能力很强，但要让它特别擅长某个具体任务（如法律文书写作、医学问答），通常需要**微调**：

> 在任务的有标注数据上，更新模型的所有参数，让模型适应这个特定任务。

### 全量微调的问题

- **存储成本巨大**：GPT-3 有 1750亿个参数，每个参数用 float16 存储就需要 350GB 显存。每个任务都需要保存一套完整的参数副本
- **训练资源要求极高**：更新1750亿个参数，每次反向传播都需要巨大计算资源
- **普通用户负担不起**：训练 GPT-3 规模的模型需要几百张 A100 GPU
- **部署困难**：有10个任务就需要10个 350GB 的模型副本，存储和切换代价极高

### 参数高效微调（PEFT）的思路

能不能只更新模型中**极少数参数**，就达到接近全量微调的效果？

这个领域叫 **参数高效微调（Parameter-Efficient Fine-Tuning, PEFT）**，LoRA 是其中最成功的方法之一。

### 什么是低秩矩阵？

在解释 LoRA 之前，需要理解**低秩**的概念：

一个矩阵的**秩（rank）**代表其包含的"独立信息量"。
- 一个 1000×1000 的矩阵最多有 1000 的秩（满秩），包含了完整的信息
- 但很多实际矩阵的秩远低于其维度——说明它可以被两个小矩阵的乘积近似（低秩分解）

例如：一个 1000×1000 的矩阵，如果秩为 8，可以近似分解为：
- 一个 1000×8 的矩阵 A
- 乘以一个 8×1000 的矩阵 B
- 参数量从 1000×1000=1,000,000 减少到 1000×8 + 8×1000 = 16,000（减少了62倍！）

## 这篇论文干了什么？

LoRA 提出了一种极为简洁优雅的大模型微调方法：

**核心假设**：
> 大模型在下游任务上的适应，本质上是在一个低维子空间中发生的。换句话说，微调需要改变的权重矩阵是低秩的。

基于这个假设，LoRA **冻结所有预训练参数**，只为每个需要适应的权重矩阵添加一对小的**低秩矩阵**进行训练。

结果：
- 可训练参数减少 **10,000 倍**（相比全量微调）
- 显存需求大幅减少
- 训练速度更快
- **性能接近甚至等同于全量微调**

---

## 怎么做到的？

### 核心思想：在权重旁边加一条"旁路"

对于大模型中的某个权重矩阵 W（比如一个注意力层中的 d×d 矩阵），LoRA 的做法是：

**预训练权重 W：冻结，不更新**

**额外添加一个低秩更新 ΔW = B × A**：
- A 是一个 **d×r** 的矩阵（r << d，r 是秩，一般取 4、8、16）
- B 是一个 **r×d** 的矩阵
- 两者都是可训练的

前向传播时：
```
输出 = W × 输入 + ΔW × 输入
     = W × 输入 + B × A × 输入
```

即：原始预训练模型的输出 + 低秩适应的输出

**初始化**：
- A 用随机高斯初始化
- B 初始化为全零
- 因此训练开始时 ΔW = B × A = 0，不影响预训练模型的初始输出

### 为什么这样有效？

研究者发现，全量微调时权重的变化量 ΔW 本身就具有低秩结构——这说明大模型的适应本质上确实发生在低维子空间中，LoRA 捕捉到了这种本质。

### 应用于哪些权重矩阵？

Transformer 中的自注意力层有四个权重矩阵：Q（查询）、K（键）、V（值）、O（输出）。

论文发现同时对 **Q 和 V** 应用 LoRA 效果最好，r=4 或 r=8 通常已经足够。

### 缩放因子 α

实践中，ΔW 会乘以一个超参数 α/r，用来控制低秩更新的幅度。常见设置是 α = r 或 α = 2r。

### 推理时的合并（零成本部署）

LoRA 最大的工程优势：推理时可以将 ΔW = BA 直接**加回**到原始权重 W 上，得到新权重 W' = W + BA，然后删掉 A 和 B 矩阵。

这意味着：
- **推理速度与原始模型完全相同**（不需要额外计算）
- **不引入任何推理延迟**，这是 Adapter 等方法的主要缺点（需要额外前向传播）

---

## 结论

1. 与全量微调 GPT-3 相比，LoRA **减少了可训练参数10,000倍，显存减少3倍**，性能基本持平
2. 在多个 NLP 基准上，LoRA 微调的效果与全量微调相当，有时超过全量微调（防止过拟合）
3. 一台消费级显卡（单张 RTX 3090）可以微调 7B 参数的模型（使用 LoRA + int4 量化）
4. LoRA 极大地民主化了大模型微调，成为开源社区微调 LLaMA 系列的标准方法
5. 各种 LoRA 变体不断涌现（QLoRA、LoHA、LoftQ 等），进一步降低了资源需求

---

## 局限性

1. **超参数选择敏感**：秩 r 和 α 的选择对效果有影响，需要调整
2. **不适合所有层**：通常只对注意力层应用 LoRA，FFN 层应用较少，可能遗漏部分适应信号
3. **多任务时需要多套 LoRA**：每个任务需要一套 LoRA 权重，管理复杂（不过比保存多套完整模型好得多）
4. **秩的选择理论依据不足**：低秩假设虽然实践有效，但严格的理论证明仍不完整
5. **数学上不完美**：低秩分解是近似，必然会丢失一些信息，对于某些需要大量专业知识的任务可能不够用

---

## 一句话总结

LoRA 通过"冻结大模型全部权重，只在每个权重矩阵旁边添加一个极小的低秩旁路"的思路，用不到1%的参数实现了接近全量微调的效果，并且推理时零延迟开销，让普通研究者用一张消费级显卡就能微调大语言模型，成为开源AI社区最常用的微调技术。
